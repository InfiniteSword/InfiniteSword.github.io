JavaScript 的继承方式

个人归纳总结，部分抄自高程三

class 继承(ES6)

Object.create实现类式继承(ES5)

(from MDN)

原型链

原型链实现继承时存在两个问题:
1、引用类型的属性会被所有实例共享。
2、创建子类型实例时不能给超类型传参。

构造函数

在子类型构造函数内部调用超类型构造函数即可解决引用类型值的问题，同时还能向超类型传参。

构造函数的问题:
在超类型原型中定义的方法，对子类型是不可见的。

组合继承

将上两种方法组合在一起的方法。

据说是JavaScript最常用的继承模式。

原型式继承

在ES5可以将object()替换成Object.create()。试用于没必要创建构造函数，只想一个对象和另一个对象保持相似的情况。和原型链模式一样，引用类型的属性会被共享。

寄生式继承

这种方法和构造函数一样不能做到函数复用。

寄生组合式继承

之前说到组合继承是JavaScript中最常用的继承模式，但是使用时至少会调用两次超类型构造函数(call一次，new一次)。而使用寄生组合继承则不用:

避免了new操作再执行一次构造函数并在子类prototype上添加一些不必要的属性。
