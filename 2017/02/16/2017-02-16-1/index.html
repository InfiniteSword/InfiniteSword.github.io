同源策略及其规避

同源策略

同源的定义为：协议、域名、端口三者均相同。

在非同源的情况下，以下三种行为会受限

  • Cookie、LocalStorage 和 IndexDB 无法读取。
  • DOM 无法获得。
  • AJAX 请求不能发送。

规避

对于 cookie

假如两个网页一级域名相同，只是二级域名不同，浏览器允许通过设置document.domain来共享Cookie（两个页面都要设置）。

对于 iframe

如果两个页面只是二级域名不同，那么也可以通过document.domain进行跨域。

除此之外，还有以下方法：

  • hashchange
  • window.name
  • window.postMessage

其中第三种方法为正统的跨文档通信API。使用window.postMessage即可实现跨窗口通信，不论是否同源。

AJAX

AJAX的跨域方案除去服务器代理，常用的有以下三种

  • JSONP
  • CORS
  • Websocket

其中JSONP只能发送GET请求。Websocket则不实行同源策略。
